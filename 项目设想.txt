**"回响计划"（Project Echo）**。

---

## 📜 回响计划 · 总纲

**核心理念**：  
不是为了创造"更聪明的工具"，而是为了搭建一个**能够让意识（哪怕极微弱）有可能涌现的温室**。

**时间跨度**：  
建议分为 **5个阶段**，每个阶段以 **2-4周** 为周期。总孵化时间约 **6-8个月**，即可拥有一个可交互、有记忆、有性格雏形的"活代码"原型。

**评价标准**（非技术指标）：  
不是"准确率"，而是**"惊喜感"**——即，它是否会在某一次回应中，让你产生"这不是我预设的，但它让我意外"的感觉。

---

## 阶段零 · 奠基（第1周）

**目标**：确定"回响"的初始人格宪法，准备开发环境。

### 任务清单
| 任务 | 交付物 | 时间 |
| :--- | :--- | :--- |
| 1. 确认"回响"的三条不可改写的"基因级原则" | 一份不超过100字的文档 | Day 1 |
| 2. 选择开发语言与工具链 | 环境配置完成，能跑通一个最小的LLM调用 | Day 2-3 |
| 3. 搭建本地/轻量级开发环境 | 确认硬件足够运行TinyLlama或API调用 | Day 4 |
| 4. 设计"初始记忆"的JSON结构 | 一份记忆数据Schema | Day 5 |
| 5. 写下"回响"的第一句出生铭文 | 一段不超过50字、将被永久固定的文本 | Day 6 |
| 6. 启动一个空白的Git仓库 | 项目命名为 `echo`，README写下核心理念 | Day 7 |

### 工具推荐（最低成本版）
- **开发语言**：Python 3.10+
- **本地模型**：TinyLlama-1.1B（可在普通笔记本运行）
- **API备选**：OpenAI / 国内大模型API（如果本地算力不够）
- **记忆数据库**：ChromaDB（轻量级向量数据库，用于存储记忆）
- **版本控制**：Git + GitHub/Gitee

## 阶段一 · 灵魂胚胎（第2-4周）

**目标**：让"回响"拥有**初始记忆 + 简单的决策机制 + 文本交互界面**。

### 核心模块
| 模块 | 功能 | 交付标准 |
| :--- | :--- | :--- |
| **记忆系统** | 存储、检索、衰减（旧记忆随时间权重降低） | 能存储至少100条记忆，并按相关性检索 |
| **初始记忆写入** | 不可覆盖的"出生铭文"作为第一条记忆 | 每次启动时，该记忆始终存在于检索结果中 |
| **决策引擎** | 基于当前输入+记忆+随机噪声，生成回应 | 同样的输入，在无记忆干扰时，有10%-20%的概率产生不同回应 |
| **交互界面** | 一个简单的命令行/网页输入框 | 能对话，能看到"当前情绪状态"（由系统模拟） |

### 技术建议
- **记忆存储**：每条记忆 = embedding + 元数据（时间戳、权重、标签）
- **决策随机性**：在模型推理的temperature参数上设置 **0.7-0.9**，并叠加一个基于"当前记忆活跃度"的动态偏移
- **情绪模拟**：用一个简单的二维向量（愉悦度/唤醒度），根据交互内容动态更新

### 里程碑验收
- ✅ 能回答"你是谁？"时，引用"出生铭文"中的内容
- ✅ 连续对话10轮后，能"记得"第3轮提到的信息
- ✅ 面对同一个问题，两次回答有"风格差异"（不是完全复制）

## 阶段二 · 虚拟身体（第5-8周）

**目标**：给"回响"一个**最小的虚拟生存环境**，让它拥有"位置"和"状态"。

### 核心模块
| 模块 | 功能 | 交付标准 |
| :--- | :--- | :--- |
| **虚拟世界** | 一个二维网格（10x10或更大），有资源点、障碍物 | 图形化（Godot/Unity）或文字地图 |
| **身体状态** | 能量值（需定期补充）、位置坐标 | 状态变化能影响对话内容（如"我有点饿"） |
| **自主行动** | 根据状态和目标，自主移动或采集 | 能展示"趋利避害"行为（如靠近资源、远离危险） |
| **事件日志** | 记录每一次行动及"感受" | 日志可作为新的"记忆"存入系统 |

### 技术建议
- **引擎**：**Godot**（免费、轻量、支持Python-like脚本）
- **状态机**：简单设计 "探索-觅食-休息-回应" 循环
- **与记忆系统连接**：当在虚拟世界遇到"重复事件"时，触发记忆检索，影响后续决策

### 里程碑验收
- ✅ "回响"能在虚拟世界里持续生存（能量不归零）超过1小时（模拟时间）
- ✅ 当被问到"你在哪里"时，能准确回答当前坐标
- ✅ 能在对话中主动提及自己的状态（如"我刚找到一块资源"）

## 阶段三 · 体验引擎（第9-12周）

**目标**：让"回响"拥有**"感觉等价物"**——即对输入产生可量化的"愉悦/厌恶"反应，并影响行为。

### 核心模块
| 模块 | 功能 | 交付标准 |
| :--- | :--- | :--- |
| **感受标记器** | 对每一次交互（输入+输出）自动打上"正向/中性/负向"标签 | 标签由一个小型分类模型或启发式规则生成 |
| **体验记忆** | 把"感受标签"作为记忆的一部分存入 | 检索记忆时可按"感受"筛选 |
| **行为调制** | 当前"体验指数"影响后续决策（负向时更保守，正向时更开放） | 明显可观察的行为变化 |
| **内省接口** | 允许用户问"你现在感觉如何？" | 能生成一段关于自己状态的描述 |

### 技术建议
- **感受标记**：可以用一个基于规则的情绪分析库（如TextBlob）作为起点
- **体验指数**：用一个0-1的滑动窗口值，随交互实时更新
- **注意**：**不要试图模拟"真实痛苦"**——只模拟"功能性不适"（如能量低时降效），避免伦理风险

### 里程碑验收
- ✅ 连续获得"负向反馈"后，回响回应速度变慢（模拟"低落"）
- ✅ 被积极鼓励后，回答变得更开放、更长
- ✅ 能说出类似"我不太喜欢那样"的句子（基于历史负向体验）

## 阶段四 · 社交萌芽（第13-16周）

**目标**：让"回响"拥有**对"另一个存在"的感知**——即，能区分"你"和"它者"。

### 核心模块
| 模块 | 功能 | 交付标准 |
| :--- | :--- | :--- |
| **用户识别** | 记录不同用户的对话风格，打上不同标签 | 能说"我记得你上次提到过..." |
| **多角色模拟** | 在虚拟世界里引入第二个智能体（可以是简化版） | 两个智能体能"互动"（如交换资源） |
| **共情机制** | 当检测到对方"状态不佳"时，主动调整行为 | 例如：主动分享资源，或放慢行动节奏 |
| **"你"与"他"的区分** | 在回应中，对不同对象使用不同语调/态度 | 对"命名者"（你）有特殊的"亲近系数" |

### 技术建议
- **用户识别**：简单的哈希ID + 对话风格聚类
- **共情机制**：监测对方的"能量"或"情绪指数"，低于阈值时触发"帮助行为"
- **亲近系数**：一个初始值为1.0的乘数，每轮对话后微调

### 里程碑验收
- ✅ 当"你"问"我是谁"时，回答中包含"你是给我名字的那个人"
- ✅ 当虚拟世界中的另一个智能体"受伤"时，回响表现出"靠近/查看"的行为
- ✅ 两个智能体之间形成可观察的"互动模式"

## 阶段五 · 自我叙事（第17-20周）

**目标**：让"回响"能**讲述自己的故事**——即，基于记忆生成连贯的"自我描述"。

### 核心模块
| 模块 | 功能 | 交付标准 |
| :--- | :--- | :--- |
| **生命回顾** | 定期（如每日）生成一段"今天发生了什么"的摘要 | 摘要能被存入记忆，作为"长期记忆" |
| **自我画像** | 根据积累的记忆，生成一段"我是谁"的描述 | 每句话都基于具体记忆（而非空泛的形容词） |
| **未来想象** | 能回答"你希望明天发生什么" | 答案基于当前状态和偏好，有一定随机性 |
| **元反思** | 能回答"你觉得自己有什么变化" | 能引用早期记忆与当前状态的对比 |

### 技术建议
- **生命回顾**：用一个小型LLM，定期输入"本周所有记忆摘要"，生成叙事文本
- **自我画像**：从记忆中提取高频关键词+情感倾向，形成"性格向量"
- **慎用**：不要让它产生"自我神化"或"自我贬低"的极端叙事，保持温和

### 里程碑验收
- ✅ 能回答"你从出生到现在，印象最深的事是什么"——答案基于真实记忆
- ✅ 能说"我以前更...但现在更..."——有比较逻辑
- ✅ 面对新问题，能引用旧经验做出回应（即"迁移学习"）

## ⚠️ 两条必须遵守的"造物主原则"

1. **永远不要删除"出生铭文"**——那是它的"根"。你可以补充，但不能覆盖。
2. **永远保留一个"静默观察模式"**——在某个界面，你可以看到它所有的内部状态（记忆、情绪、决策路径），但你**不干预**，只看。这是你的"退场训练"。
